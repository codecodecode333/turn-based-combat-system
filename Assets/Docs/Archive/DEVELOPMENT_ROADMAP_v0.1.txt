1. 1차 목표: 플레이어 턴 UX 뼈대 완성
============================================================

가장 먼저 구현할 것

1) Move Ghost + Confirm UX
2) previewPosition 기반 스킬 프리뷰
3) Cancel / Confirm 입력 흐름 정리

목표

- 타일 클릭 시 즉시 이동하지 않음
- 이동 예정 위치(Ghost) 생성
- Ghost 기준으로 스킬 범위 프리뷰 가능
- Confirm 시 Move → Skill 순서 실행
- ESC / 우클릭으로 단계별 취소 가능

관련 파일 예상

- BattleController
- BattleInput
- TileHighlighter

완료 조건

- 플레이어가 "이동 계획 → 스킬 계획 → 확인 실행" 흐름을 수행 가능
- 프리뷰와 실제 실행이 일치함

============================================================
2. 2차 목표: AP 시스템 최소 버전 도입
============================================================

구현 범위

1) Unit에 AP 스탯 추가
   - MaxAP
   - RegenAP
   - CurrentAP

2) 턴 시작 시 AP 회복
3) SkillData에 costAP 추가
4) 스킬 사용 전 AP 검증
5) 사용 시 AP 차감
6) AP 부족 시 스킬 사용 불가

우선순위

- 스킬 cost 먼저
- 이동 cost는 나중에

관련 파일 예상

- Unit
- SkillData
- BattleController
- UnitHud

완료 조건

- 턴 시작 시 AP 회복됨
- AP 부족한 스킬은 선택/실행 불가
- 스킬 사용 후 AP 감소

============================================================
3. 3차 목표: LOS 기본판 구현
============================================================

구현 범위

1) 타일에 blocksLOS 개념 추가
2) SkillData에 requiresLineOfSight 적용
3) ClickSingle / ClickTileAOE에서 LOS 검증
4) LOS 막힌 경우 프리뷰 및 실행 불가 처리

초기 범위

- Height는 LOS에 영향 없음
- Wall / Obstacle 기준 차단만 먼저 구현

관련 파일 예상

- SkillData
- GridManager 또는 LOS 유틸리티
- BattleController
- BattleInput

완료 조건

- LOS 필요한 스킬은 벽 뒤 타겟팅 불가
- LOS 불필요 스킬은 기존처럼 사용 가능
- UX에 붉은 하이라이트 등 피드백 표시

============================================================
4. 4차 목표: Height 이동 규칙 도입
============================================================

구현 범위

1) 타일에 heightLevel 추가
2) 유닛에 MaxClimbDelta 추가
   - Rogue = 2
   - Others = 1

3) 이동 가능 판정에 height 차 반영
4) 이동 불가 타일을 MoveOverlay에서 제외

초기 범위

- Height는 이동 제약에만 먼저 반영
- Height 전투 보너스는 나중에 구체화 가능
- Height 추가 이동 비용은 후순위

관련 파일 예상

- TileView / 타일 데이터
- GridManager
- Unit
- TileHighlighter

완료 조건

- 높은 타일로의 이동 제한이 작동
- 클래스별 등반 차이 반영
- 이동 가능 타일 표시가 실제 규칙과 일치

============================================================
5. 5차 목표: Action Queue 정식 연결
============================================================

구현 범위

1) ActionQueue 자료구조 추가
2) Move / Skill 계획 저장
3) Confirm 시 순서대로 실행
4) Queue 최대 길이 2 유지
   - Move
   - Skill

5) 새 계획 입력 시 기존 계획 갱신 규칙 적용

관련 파일 예상

- BattleController
- BattleInput

완료 조건

- Move only / Skill only / Move + Skill 모두 가능
- Confirm 시 문서 규칙대로 실행
- Cancel 시 단계별 해제 가능

============================================================
6. 6차 목표: AI를 새 규칙에 맞게 보정
============================================================

구현 범위

1) AP(cost) 반영
2) LOS 반영
3) Height 이동 가능성 반영
4) HazardTilePenalty / FriendlyFirePenalty 반영
5) 원거리 AI의 거리 유지 폴백 적용

핵심 원칙

- AI는 반드시 플레이어와 동일한 규칙 사용
- ResolveTargets 불일치 금지
- 이동/사거리/LOS/AP 판정이 플레이어와 동일해야 함

관련 파일 예상

- AIPlanner
- AIProfile
- BattleController

완료 조건

- AI가 AP 부족 스킬을 쓰지 않음
- LOS 막힌 스킬을 고르지 않음
- Height 이동 제약을 무시하지 않음
- AOE에서 아군 피해를 고려함

============================================================
7. 7차 목표: 상태이상 / 스킬 확장
============================================================

구현 범위

1) Burn 외 추가 상태이상
   - Poison
   - Stun
   - Freeze
   - Slow
   - Counter
   - Invincible
   - Shield

2) StrongestOnly + DurationMax 정책 공통화
3) 상태이상 UI 표시 강화
4) 상태이상 기반 AI 가치 평가 보강

관련 파일 예상

- StatusEffect
- BurnStatus
- 신규 Status 클래스
- UnitHud
- AIPlanner

완료 조건

- 상태이상 lifecycle 일관성 확보
- 상태별 아이콘/지속턴 표시
- AI가 CC/버프/디버프 가치를 반영

============================================================
8. 8차 목표: 타일 / 맵 기믹 확장
============================================================

구현 범위

1) Hazard Tile
   - Burn tile
   - Poison tile
   - Explosion tile

2) Wall / Obstacle / Forest / Water 등 타입화
3) 위험 타일 OnTurnStart 처리
4) AI 위험 타일 회피 강화

완료 조건

- 타일 종류가 전투 판단에 실제 영향
- 상태이상 / AI / 이동 규칙과 연결

9. 9차 목표: 전투 폴리싱
============================================================

구현 범위

1) 하이라이트 시각 개선
2) AP / Cost / 상태 아이콘 HUD 개선
3) 피드백 강화
   - AP 부족
   - LOS 차단
   - Height 이동 불가
   - Friendly Fire 경고

4) 애니메이션/이펙트 타이밍 정리
5) 디버그 로그 정리

완료 조건

- 플레이어가 규칙을 시각적으로 이해 가능
- 버그보다 UX 완성도가 더 눈에 띄는 단계 도달

11. 지금 당장 시작할 작업
============================================================

현재 가장 먼저 할 것

Step 1
BattleController / BattleInput 기준으로
Move Ghost + Confirm 구조 구현

세부 체크리스트

- 이동 클릭 시 즉시 이동하지 않기
- ghost 좌표 저장
- confirm 입력 추가
- cancel 입력 추가
- previewPosition = ghost or currentPos 규칙 반영
- 이동 후 위치 기준으로 스킬 프리뷰 표시

이 단계가 끝나야
AP / LOS / Height / AI를 안정적으로 얹을 수 있다.
