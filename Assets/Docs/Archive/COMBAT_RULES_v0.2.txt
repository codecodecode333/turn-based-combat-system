COMBAT_RULES.txt
(Isometric Tactical Combat — Combat Rules)

버전: Draft v0.2 (2026-03-05)
변경점: Height(고저) 전투 영향 추가, 강제이동/낙하 규칙 보강

============================================================
1) Damage Formula
============================================================
FinalDamage = BaseDamage + Attack - Defense
- FinalDamage 최소 1 보장 (MinDamage = 1)
- HP는 0 clamp (HP < 0 불가)

============================================================
2) Critical Hit
============================================================
- 치명타 Enabled
- critChance / critMultiplier 사용
- Crit 발생 시 FinalDamage *= critMultiplier

============================================================
3) HP / Death
============================================================
- HP == 0 이면 즉시 제거(기본)
  (옵션으로 Down 시스템을 결합 가능)

============================================================
4) Skill Use Validity (실패 조건)
============================================================
스킬은 조건 미충족 시 “사용 불가(시도 안 됨)”
- Range(min/max)
- LOS(스킬이 requiresLOS일 때)
- AP(cost) 충분

============================================================
5) Friendly Fire / Targeting
============================================================
- SingleTarget: 적/아군/오브젝트 등 설계대로 가능 (Heal/Buff/기믹 단일공격 포함)
- AOE: 범위 내 모두(아군 포함) 타겟, 중심 타일 포함
- AOE 범위: Manhattan Disk 유지

============================================================
6) Status Tick Timing / Stacking
============================================================
- Status tick: OnTurnStart
- Stacking: StrongestOnly + DurationMax (이미 합의된 정책 기준)

============================================================
7) Height Rules (Combat Interaction)
============================================================
v0.2에서는 Height가 전투 수치에 영향을 준다. (A1 = D)

정의
- 각 타일은 heightLevel을 가진다.
- 공격/피격 시 heightDelta = attackerHeight - targetHeight

권장 기본(수치 템플릿, 밸런스 단계에서 조정 가능)
- 고지대 공격 보너스(공격자 높음):
  - HitBonus 또는 DamageBonus 중 1개를 택해 적용 (초기엔 HitBonus 권장)
- 고지대 방어/회피 보너스(방어자 높음):
  - AttackerHitPenalty 또는 DamageReduction 중 1개를 택해 적용

[초기 권장안(구현 쉬움)]
- heightDelta > 0  => attackerHit +H
- heightDelta < 0  => attackerHit -H
- H는 1~2 수준의 작은 값으로 시작

주의
- “Height가 이동에만 영향”이었던 v0.1과 달리, v0.2부터 전투 수치 연동이 생김.
- 정확한 Hit 시스템이 아직 없다면, 우선 “DamageBonus/Reduction”으로 단순 구현해도 됨.

============================================================
8) Forced Movement vs Height (Push/Knockback)
============================================================
강제이동은 Height 제약을 부분적으로 따른다.

규칙
- 더 높은 타일로는 밀려날 수 없다.
  (target.heightLevel > current.heightLevel 방향으로 push 불가)
- 같거나 낮은 타일로는 밀려날 수 있다.
  (낙하/추락 방향 이동 허용)

즉,
- dh = nextHeight - currentHeight
- dh > 0  => knockback blocked
- dh <= 0 => knockback allowed (단, passable/occupied 등 기본 이동 규칙은 유지)

[낙하 데미지]
- v0.2에서는 낙하 데미지 시스템은 사용하지 않는다(추후 도입).

============================================================
9) AOE vs Height
============================================================
- v0.2에서는 AOE는 Height를 무시한다.
  (AOE 범위/대상 선정은 2D 규칙대로 처리)
- 단, 피해 계산에서 “공격자-타겟 heightDelta 보정”은 적용될 수 있다.
  (AOE casterHeight 기준으로 개별 타겟에 동일 규칙 적용 가능)
