# 전술 그리드 로그라이크 — 코어 디자인 문서 (초안)

---

# 1. 게임 개요

## 장르

* Tactical Grid Roguelike
* 아이소메트릭 그리드 턴제 전투
* 소규모 파티 전략 게임

## 핵심 경험

플레이어는 **3명의 캐릭터로 구성된 파티**를 조작하여
**아이소메트릭 그리드 기반 전투**를 진행한다.

전투는 다음 요소를 중심으로 진행된다.

* 위치 선정 (Positioning)
* 스킬 사용
* 전술적 의사결정

각 전투 이후 플레이어는 **로그라이크 업그레이드**를 선택하며,
이를 통해 플레이 도중 **캐릭터 빌드가 점진적으로 발전**한다.

## 디자인 방향

* 전술 중심 전투
* 캐릭터 빌드 시스템
* 반복 플레이 가능한 로그라이크 진행 구조

이 게임은 **퍼즐 중심이 아니라 전투 중심 게임**이다.

---

# 2. 전투 시스템

## 파티 구성

플레이어 파티

* 3명의 캐릭터

적 유닛

* 전투당 4~6명

권장 전투 밀도

```
Player: 3
Enemy: 5
```

---

## 턴 시스템

각 유닛의 턴에서 가능한 행동

```
Move
+
Use 1 Skill
```

규칙

* 이동 후 스킬 사용
* 이동은 선택 사항
* 턴당 **스킬은 1개만 사용 가능**

---

## 캐릭터 상태

캐릭터는 두 가지 상태를 가진다.

```
Alive
Down
```

### Alive

* 정상 전투 상태
* 이동 및 스킬 사용 가능

### Down

발동 조건

```
HP = 0
```

효과

* 행동 불가
* 타겟팅 불가
* 전투에서 제거된 것으로 간주

### 전투 종료 후

Down 상태 캐릭터

```
Revive
HP 일부 회복
```

이를 통해 **전투 중 긴장감은 유지하면서 영구적인 캐릭터 손실은 방지한다.**

---

# 3. 맵 디자인

## 맵 크기

권장 전투 맵 크기

```
10 x 10
~
12 x 12
```

큰 맵은 다음 요소를 강화한다.

* 포지셔닝 전략
* 사거리 이점
* 전술적 이동

---

## 적 배치

기본 규칙

```
모든 적은 전투 시작 시 등장
```

장점

* 전술 계획이 명확해짐
* 전투 템포 안정화
* AI 밸런싱이 쉬워짐

---

## 증원 시스템

**특수 전투에서만 사용**

예시

* 보스 전투 기믹
* 이벤트 전투
* 시간 기반 매복

기본 규칙은 아님.

---

## 전투 시작 거리

권장 배치 거리

```
플레이어 ↔ 적
5 ~ 7 타일
```

이 구조는 다음 흐름을 만든다.

* 1턴 : 위치 선정
* 2턴 : 교전 시작

---

# 4. 승리 조건

## 기본 승리 조건

```
Enemy Elimination
```

모든 적 처치 → 승리

---

## 목표 기반 미션

일부 전투는 **목표 기반 승리 조건**을 사용한다.

예시

* 탈출 (출구 타일 도달)
* 생존 (X턴 버티기)
* 오브젝트 파괴
* 유닛 호위
* 특정 지역 점령

규칙

```
목표 달성
→ 전투 즉시 종료
```

이러한 전투는 **특수 노드 타입**으로 등장한다.

---

# 5. 로그라이크 진행 시스템

게임 진행은 **노드 기반 맵** 구조를 사용한다.

각 전투 이후 플레이어는 **다음 노드를 선택**한다.

### 노드 종류

```
Combat
Objective Combat
Elite
Shop
Rest
Event
```

---

## 전투 후 보상

전투 승리 후 플레이어는 다음 중 선택한다.

```
3개 중 1개 선택
```

보상 종류

* 스킬 업그레이드
* 패시브 능력
* 아이템

---

# 6. 캐릭터 시스템

## 초기 캐릭터 수

계획된 최소 캐릭터 수

```
6 characters
```

향후 확장 가능.

---

## 캐릭터 능력 구조

각 캐릭터는 다음 능력을 가진다.

```
Passive ×1
Skill ×2
Ultimate ×1
```

총 능력 수

```
4
```

---

## 캐릭터 정체성

각 캐릭터는 **하나의 핵심 테마**를 중심으로 설계된다.

예시

| 캐릭터       | 테마               |
| --------- | ---------------- |
| Lavien    | Fire / Burn Mage |
| Sniper    | 장거리 딜러           |
| Guardian  | 탱커               |
| Alchemist | 독 제어             |
| Priest    | 힐러               |
| Berserker | 근접 딜러            |

캐릭터 테마는 **명확하고 직관적이어야 한다.**

---

## 캐릭터 디자인 규칙

허용되는 특징

* 강한 시각적 테마
* 명확한 전투 역할

예시

```
Burn
Poison
Range
Tank
Support
```

권장되지 않는 설계

* 핵심 전투 규칙 변경
* 턴 시스템 변경
* 이동 시스템 변경

캐릭터는 **게임 시스템을 변경하지 않으면서도 개성을 가져야 한다.**

---

# 7. 캐릭터 역할

권장 역할 분포

```
Tank
Bruiser
Ranger
Mage
Support
Control
```

예시 구성

```
Guardian (Tank)
Berserker (Bruiser)
Sniper (Ranger)
Lavien (Mage)
Priest (Support)
Alchemist (Control)
```

이 구조는 **다양한 파티 조합을 가능하게 한다.**

---

# 8. 스킬 업그레이드 시스템

플레이 도중 스킬은 **진화할 수 있다.**

예시

Fireball 업그레이드

```
A
AOE 반경 +1

B
Burn 상태 적용

C
Damage +40%
```

이 시스템은 **플레이마다 다른 빌드 경로를 만든다.**

---

# 9. 전술 맵 요소

큰 맵은 다양한 지형 요소와 잘 어울린다.

추천 지형

```
Obstacle
Wall
Forest
Water
```

위험 지형도 존재할 수 있다.

예시

```
Burn tile
Poison tile
Explosion tile
```

이 요소들은 **상태이상 시스템과 상호작용한다.**

---

# 10. 캐릭터 시각적 정체성

캐릭터는 **강한 시각적 테마**를 가질 수 있다.

예시

Lavien

```
Fire Mage
Burn specialization
```

강한 시각적 정체성은 다음을 개선한다.

* 캐릭터 기억성
* 게임 플레이 가독성
* 캐릭터 인식

---

# 11. 현재 프로젝트에서 구현된 핵심 시스템

현재 프로젝트에는 다음 시스템이 이미 구현되어 있다.

* Grid BFS 이동
* SkillTargetMode 타겟팅 시스템
* Hover AOE Preview
* Tile Highlight 시스템
* Utility 기반 AI
* StatusEffect 시스템
* SkillEffect 시스템
* Animation Event 기반 스킬 타이밍 처리

이 시스템들은 현재 디자인 방향과 **완전히 호환된다.**

---

# 12. 핵심 게임 플레이 루프

```
노드 선택
↓
전투
↓
승리
↓
로그라이크 업그레이드 선택
↓
다음 노드 이동
```

플레이어는 런(run) 동안 **전투 빌드를 점진적으로 완성한다.**

---

# 13. 아직 완전히 설계되지 않은 시스템

다음 시스템은 향후 설계가 필요하다.

* 상태 효과 프레임워크
* 아이템 시스템
* 엘리트 적 설계
* 보스 메커니즘
* 지형 상호작용 시스템

이 요소들이 게임 깊이를 크게 결정할 것이다.
