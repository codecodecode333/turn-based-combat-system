# GAME_DESIGN_DOCUMENT v0.2

## 1. 게임 개요

### 1.1 한 줄 정의

```plaintext
느리지 않고 속도감 있는, 파괴감과 뽕맛을 중심으로 한 턴제 전술 로그라이크
```

---

## 1.2 장르

* 턴제 전술
* 로그라이크
* 퍼즐 요소 포함

---

## 1.3 핵심 특징

* 빠른 템포의 턴제 전투
* 연속 스킬 사용 기반의 폭발적 플레이
* 상태이상 + 타일 + 위치의 상호작용
* 빌드에 따라 완전히 달라지는 전투 스타일

---

# 2. 코어 게임 루프

```plaintext
전투 → 보상 선택 → 강화 → 다음 전투
```

## 2.1 전투

* 적과 턴제 전투 진행
* 위치 / 스킬 / 상태이상 활용

## 2.2 보상

* 스킬 선택
* 스킬 강화
* 패시브 강화

## 2.3 성장

* 빌드 방향 형성
* 시너지 조합 생성

---

# 3. 핵심 재미

## 3.1 터지는 쾌감

* 강한 스킬
* 다수 적중
* 연쇄 효과
* 상태이상 폭발

## 3.2 빌드 성장

* 스킬 조합 변화
* 강화 선택에 따른 플레이 변화
* 매 판 다른 전략

## 3.3 전술 성공의 쾌감

* 위치 선택
* 범위 계산
* 리스크 판단
* 한 턴 최적 플레이

---

# 4. 전투 스타일

```plaintext
액션적 체감 60% + 전술적 판단 40%
```

## 4.1 특징

* 빠른 의사결정
* 직관적인 결과
* 깊이는 조합에서 발생

## 4.2 목표

* “빠르지만 얕지 않은 전투”
* “생각은 하지만 답답하지 않음”

---

# 5. 전투 시스템 핵심

## 5.1 행동 구조

```plaintext
Move (0~1회)
Skill (AP 기반 N회)
```

* 이동 후 스킬 사용
* AP가 허용하는 만큼 연속 행동 가능
* 한 턴 내 폭발적 플레이 가능

---

## 5.2 Preview 시스템

```plaintext
Preview = Execution
```

* 결과를 미리 확인 가능
* 예측 가능한 전투
* 실수는 전략의 결과

---

## 5.3 상태이상 시스템

* Burn / Poison / Stun / Freeze / Slow / Counter / Shield / Invincible
* 상태는 큐 방식으로 적용
* 전투 흐름에 큰 영향

---

## 5.4 타일 시스템

* Height 기반 이동 전략
* Hazard (Burn / Poison / Explosion)
* LOS 차단

---

# 6. 로그라이크 시스템

## 6.1 구성 요소

* 스킬 획득
* 스킬 강화
* 패시브
* 아이템

---

## 6.2 방향

```plaintext
플레이어 선택 기반 빌드 생성
```

* 안정형 / 폭발형 선택 가능
* 빌드에 따라 플레이 방식 변화

---

# 7. 플레이 자유도

```plaintext
유도형 자유
```

* 다양한 플레이 가능
* 시스템이 자연스럽게 효율적인 선택 유도

---

# 8. 난이도 체감

```plaintext
조금 억울하지만 납득 가능한 실패
```

## 8.1 설계 원칙

* 대부분 실패는 플레이어 선택의 결과
* 랜덤 요소 최소화
* 예측 가능한 전투

---

# 9. 캐릭터 설계

## 9.1 구조

```plaintext
Passive 1
Unique Skill 2
Common Skill 1
```

## 9.2 특징

* 캐릭터별 플레이 방식 차이 큼
* 빌드 방향에 따라 완전히 다른 플레이

---

# 10. 전투 템포

## 10.1 턴 체감

```plaintext
빠르지만 판단이 필요한 템포
```

## 10.2 전투 길이

* 기본: 3~8턴
* 강한 빌드: 더 짧아짐
* 어려운 상황: 길어질 수 있음

---

# 11. 연출 / 피드백

## 11.1 중요도

```plaintext
중간 ~ 높음
```

## 11.2 방향

* 스킬 임팩트 강조
* 상태이상 시각화
* 타격감 강조
* 과하지 않지만 확실한 피드백

---

# 12. 스트레스 설계

```plaintext
낮음 ~ 중간
```

## 12.1 방향

* 쾌감 중심
* 과도한 페널티 없음
* 실패는 학습으로 연결

---

# 13. 차별점

```plaintext
판타지 기반의 빠르고 파괴감 있는 턴제 전투
```

## 핵심 차별 요소

* 턴제지만 느리지 않음
* 한 턴 내 연속 행동 구조
* 상태이상 + 타일 + 스킬 시너지
* 로그라이크 빌드 기반 폭발력

---

# 14. 설계 철학

## 14.1 단순 + 조합

* 규칙은 단순하게
* 조합으로 깊이 생성

## 14.2 예측 가능성

* 결과를 항상 예상 가능
* 랜덤 최소화

## 14.3 속도감

* 불필요한 입력 제거
* 빠른 턴 진행

## 14.4 파괴감

* 강한 스킬
* 연쇄 효과
* 시각적 피드백
