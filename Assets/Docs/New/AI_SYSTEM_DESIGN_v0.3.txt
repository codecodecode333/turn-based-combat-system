# AI_SYSTEM_DESIGN v0.3

## 목적

본 문서는 전투 AI의 행동 원칙, 평가 구조, 난이도 조절 방식, 성향 분화를 정의한다.

AI는 플레이어와 동일한 전투 규칙을 사용해야 하며,
다음 요소를 함께 고려해 행동을 선택한다.

* 이동
* 공격
* 상태이상
* 위험 회피
* AP 효율
* Friendly Fire
* Hazard
* 턴 내 연속 행동 가치

현재 AI는 후보 생성 → 점수 계산 → 최고 선택 구조를 사용하며, 이 구조는 유지한다.

---

# 1. 핵심 원칙

## 1.1 동일 규칙 원칙

AI는 플레이어와 동일한 규칙을 따라야 한다.

* Range
* LOS
* AP
* Height 이동 가능성
* Hazard
* Friendly Fire
* 타겟 판정

즉, Preview / Player / AI는 같은 타겟 해석 규칙을 공유해야 한다.

## 1.2 목표

AI의 최종 목표는 “항상 최대 데미지”가 아니라,
**현재 턴과 다음 턴까지 포함한 전투 가치 최대화**다.

---

# 2. AI 성향 구조

AI는 전역 단일 성향이 아니라 **유닛별 성향 차이**를 가진다.

```plaintext id="ai_tone"
공격형
균형형
생존형
제어형
원거리 유지형
```

## 2.1 공격형

* 데미지 / 킬 가치 우선
* 위험 감수 성향 높음
* Overkill 허용 범위 큼

## 2.2 균형형

* 데미지와 생존을 함께 고려
* 표준 AI 기본값으로 적합

## 2.3 생존형

* Hazard / 위협 회피 우선
* 무리한 진입 억제
* 방어 스킬과 안전 위치 선호

## 2.4 제어형

* CC / Debuff / Utility 가치 높게 평가
* 킬보다 흐름 통제 선호

## 2.5 원거리 유지형

* 공격 가능 + 안전을 최우선
* 적당 거리 유지
* 무리한 접근 억제

---

# 3. 난이도 스타일

AI는 “약간 인간처럼 실수하는 정직한 AI”를 기본으로 한다.

## 3.1 기본 철학

* 완전 최적 AI는 플레이어에게 무력감을 줄 수 있다.
* 약간의 비최적 선택은 전투를 더 자연스럽고 덜 억압적으로 만든다.
* 난이도 상승은 단순 치팅보다 **실수 감소 + 판단 개선**으로 주는 것이 좋다.

## 3.2 난이도 기본 방향

```plaintext id="ai_diff"
쉬움   = 실수율 높음 / 보수적
보통   = 균형형 / 약간의 실수
어려움 = 실수율 낮음 / 판단력 상승
```

## 3.3 구현 방향

* mistakeChance 유지
* topK 후보 중 선택 구조 유지
* 난이도에 따라 topK 범위와 실수율 조절 가능

---

# 4. 행동 결정 구조

현재 구조를 유지한다.

```plaintext id="ai_loop"
후보 생성
→ 점수 계산
→ 최고 후보 선택
```

이 방식은 이미 다음 요소를 함께 평가할 수 있다.

* 피해량
* 처치 가치
* 상태이상 가치
* Friendly Fire
* Hazard
* AP 비용
* 이동 위험도
* 위치 이득

현재 AIPlanner는 이 구조를 기반으로 동작한다.

---

# 5. 이동 vs 공격 우선순위

AI는 “항상 공격”도 아니고 “항상 위치 우선”도 아니다.
상황 기반으로 판단한다.

## 5.1 기본 원칙

```plaintext id="ai_move_attack"
공격 가능 + 안전하면 우선
공격 가능하지만 지나치게 위험하면 재평가
공격 불가면 다음 턴 각을 만드는 이동 선택
```

## 5.2 해석

* 이번 턴에 유효한 공격이 가능하면 높은 우선순위
* 단, Hazard나 과도한 노출은 감점
* 아무 공격도 안 되는 턴에는 거리/지형/안전이 좋은 위치를 선호

---

# 6. 타겟 선택 원칙

타겟 선택은 고정 우선순위가 아니라 상황 기반으로 평가한다.

## 6.1 고려 요소

* 남은 HP
* 예상 킬 가능성
* 현재 위협도
* 거리
* 상태이상 적용 가치
* 반격 위험
* 무적 / 실드 / 카운터 보유 여부

현재 AI는 낮은 HP, 카운터, 실드, 무적 등을 이미 평가에 반영한다.

## 6.2 타겟 원칙

```plaintext id="ai_target"
킬 가능 대상 우선
그다음 고위협 대상
그다음 총 기대가치가 높은 대상
```

---

# 7. Overkill 정책

AI는 Overkill을 절대 금지하지 않지만, 가능한 한 최소화한다.

## 7.1 원칙

* 처치가 중요하면 일부 Overkill 허용
* 지나친 Overkill은 감점
* 낮은 HP 적에게 과한 고비용 스킬 남발은 억제

## 7.2 이유

* 플레이어 입장에서 AI가 너무 정확하게 낭비 없이만 움직이면 기계적으로 느껴질 수 있음
* 완전 허용하면 스킬 낭비가 심해짐

---

# 8. Hazard 판단

Hazard는 회피 대상이지만, 절대 규칙은 아니다.

## 8.1 기본 정책

```plaintext id="ai_hazard"
이득이 더 크면 감수 가능
기본은 회피
```

## 8.2 우선순위

* 치명적 Hazard는 강하게 회피
* 낮은 위험은 공격 가치가 더 크면 감수 가능
* 저체력일수록 회피 성향 강화

현재 HazardUtility 기반 감점 구조와 잘 맞는다.

---

# 9. Friendly Fire 정책

Friendly Fire는 무조건 회피도, 무조건 허용도 아니다.
계산 후 허용하는 방향이 맞다.

## 9.1 원칙

```plaintext id="ai_ff"
아군 피해 < 전술적 이득 이면 허용 가능
```

## 9.2 예시

* 적 3명 타격 + 아군 1명 경미한 피해 → 가능
* 적 1명 타격 + 아군 큰 피해 → 비권장

현재 AI는 Friendly Fire penalty를 이미 계산한다.

---

# 10. 상태이상 가치

상태이상은 “있으면 좋은 옵션” 정도가 아니라, **상황 기반의 고가치 요소**다.

## 10.1 원칙

* DOT는 누적 기대값으로 평가
* CC는 전투 흐름 제어 가치로 평가
* 방어형 상태는 생존 가치로 평가

## 10.2 상태별 해석

* Burn / Poison = 지속 피해
* Stun / Freeze = 강력 제어
* Slow = 위치 통제
* Counter = 반격 위협
* Shield = 생존
* Invincible = 강력 방어

현재 AI는 상태이상 가치와 기존 상태 중복 감가까지 일부 반영한다.

---

# 11. CC 우선순위

CC는 항상 최우선이 아니다.
상황 따라 가치가 크게 오르는 요소다.

## 11.1 원칙

```plaintext id="ai_cc"
킬이 더 좋으면 킬 우선
CC가 더 좋으면 CC 우선
```

## 11.2 이유

* AI가 지나치게 정확하게 CC만 반복하면 플레이어 무력감이 커진다
* 매 턴 최적 CC만 선택하면 전투 패턴이 단조로워진다
* 공격형 AI와 제어형 AI의 차이를 유지할 필요가 있다

## 11.3 제어형 AI 예외

* 제어형 성향은 CC 가치 가중치를 더 높게 둘 수 있다

---

# 12. 거리 유지 전략

원거리 AI는 “최대 거리 유지”보다 **상황 기반 거리 유지**가 좋다.

## 12.1 원칙

```plaintext id="ai_range"
공격 가능 + 안전 우선
그다음 적당 거리 유지
```

## 12.2 이유

* 지형, LOS, Hazard 때문에 무조건 최대 거리만 고집하면 멍청해진다
* 공격 기회를 버리지 않도록 해야 한다

이 방향은 현재 AI 문서 및 코드 방향과 일치한다.

---

# 13. 위치 전략

위치 선택은 완전 랜덤도, 완전 수학 최적화도 아니다.
성향별 차이를 둔다.

## 13.1 성향별 차이

* 공격형: 위험 감수
* 생존형: 안전 우선
* 원거리형: 각 유지
* 제어형: 다수 적중 위치 선호

## 13.2 랜덤성

* 완전 랜덤 이동은 금지
* topK 후보 중 선택으로 자연스러운 변형 허용
* 난이도에 따라 랜덤성 감소 가능

---

# 14. 스킬 선택 기준

AI는 최대 피해만 보지 않는다.
**총 기대가치 최대화**가 기준이다.

## 14.1 평가 요소

```plaintext id="ai_value"
Damage
Kill
Status
TargetCount
FriendlyFire
Threat
Hazard
Cost
```

이 구조는 현재 구현과 일치한다.

## 14.2 해석

* 단일 큰 피해보다 다수 타겟 + 상태이상이 나을 수 있다
* 킬 확정은 큰 보너스
* 비용이 큰 스킬은 기회비용 감점
* 위험 노출이 과하면 공격 가치 일부 상쇄

---

# 15. 스킬 다양성

AI는 항상 하나의 정답 스킬만 반복하지 않도록 성향 차이를 둔다.

## 15.1 정책

```plaintext id="ai_variety"
다양성 = 성향 기반
```

## 15.2 예시

* 공격형: 강한 단일기 반복 가능
* 제어형: 상태이상/디버프 선호
* 균형형: 피해/위치/상태 조합
* 쉬움 난이도: 약간 비최적 선택 허용

---

# 16. 난이도 조절

난이도는 수치와 판단력을 함께 조절한다.

## 16.1 수치 조절

* HP
* 공격력
* 방어력
* AP
* 상태 수치

## 16.2 판단력 조절

* mistakeChance
* topK
* hazard 회피 강도
* friendly fire 허용선
* overkill 억제 정도

## 16.3 원칙

```plaintext id="ai_scale"
난이도 = 수치 + 판단력
```

---

# 17. 현재 유지할 것 / 확장할 것

## 유지

* 후보 생성 → 점수 계산 → 선택
* Friendly Fire 감점
* Hazard 감점
* AP 비용 감점
* 상태이상 가치 반영
* topK / mistakeChance 기반 자연스러운 오차

## 확장

* 성향별 가중치 차이 강화
* 스킬 카테고리 태그 읽기
* self-target 가능 여부 반영
* 패턴형 스킬 해석
* 난이도별 판단 편차 조절

---

# 18. 리팩토링 방향

현재 AI에서 명확히 제거할 요소는 없다.
대신 다음을 점진적으로 정리하면 된다.

* 하드코딩된 상태 가치 일부를 프로필 데이터로 승격
* 스킬 카테고리/태그 기반 평가 강화
* 행동 로그 디버깅 출력 개선
* 성향별 프로필 프리셋 분리

---

# 19. 핵심 요약

```plaintext id="ai_summary"
- AI는 유닛별 성향 차이를 가진다
- 기본은 약간 인간처럼 실수하는 정직한 AI
- 행동 결정 구조는 후보 생성 + 점수 계산 유지
- 공격/이동/상태/위험은 상황 기반으로 함께 판단
- CC는 항상 최우선이 아니라 상황 따라 고가치
- 난이도는 수치 + 판단력으로 조절
```
