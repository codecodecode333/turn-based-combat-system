# 핵심 전투 구조 (확정)

```plaintext
Move (0~1회, AP 없음)
Skill (AP 기반 N회)
```

* Move → Skill만 가능
* Skill 이후 Move 불가
* AP가 허용하는 한 여러 스킬 사용 가능
* Confirm 시 ActionQueue 실행

# 전투 UX 핵심 원칙

* Preview = Execution 반드시 동일
* Hover = 항상 Selected
* Invalid Overlay 없음 (메시지 기반 피드백)
* Target Lock > Hover
* Base + FX Overlay 구조 유지
* 상태이상 = 월드 이펙트 중심
